## 3. 关于"机器日志算不算泄密"的判断

你的疑问：机器卖给医院后，医院能从软件下载 log，那这些 log 还算机密吗？

**判断：在你这个用途下，仍按机密处理。** 理由：

1. **"医院能下载自己设备的 log" ≠ "你能把跨客户的汇总数据公开"。** 单台设备的 log 归该医院；你手上的是**跨 100+ 台、跨客户的汇总数据集**——这个**集合**是西门子的专有资产。
2. **log 内的参数体系本身是专有的**：参数命名、阈值、采集逻辑、版本演进，是研发 know-how，不是公开规格。
3. **可反推性**：log + 序列号 + 时间 + 地点能反推到具体客户/设备。
4. **所有权不因脱敏改变**：删 PII 不改变"这是西门子 MRI 数据集"的事实。

> 一句话：**单客户能看自己的 log，不等于你能公开跨客户数据集。** 风险不值得赌——尤其你还在职 + 要走 F-1→OPT→H-1B/绿卡。

## 4. 为什么"脱敏后发布"仍不安全（核心论据）

1. **自由文本脱敏不可靠**：narrative 里嵌着序列号片段、版本、参数、错误码、写法习惯，组合起来可被反推（学界共识：非结构化文本"匿名化"经常可逆）。
2. **数据集本身就是 IP**：与 PII 无关，"西门子 MRI 工单的内容/故障分布/处理方式"本身即商业机密。
3. **在职 + 签证**：与前雇主因商业秘密起争议，对身份规划是灾难级风险。
4. **作品集不需要真实数据**：招聘官看的是"你会不会建这套系统"，不是"你有没有西门子数据"。
